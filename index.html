802.1X Configuration Generator

Configuration Options

Select the specific operating system or platform version. Different platforms from the same vendor may have different configuration syntax and capabilities.
Choose the authentication method to use:
  • 802.1X Only: Authenticates devices using 802.1X protocol only
  • MAB Only: Authenticates devices by MAC address only
  • 802.1X + MAB: Tries 802.1X first, then falls back to MAB if 802.1X fails
  • Multi-Auth: Allows multiple authentication methods simultaneously
  • Multi-Domain: Separate authentication for voice and data devices
  • Multi-Host: Authenticate one device and allow multiple others on same port
IP address of the primary RADIUS authentication server. This server will be tried first for all authentication requests.
Shared secret used to encrypt RADIUS communications. Should be a strong, unique key at least 16 characters long with mixed case, numbers, and special characters.
IP address of the backup RADIUS server. Used if the primary server is unreachable.
Shared secret for the backup RADIUS server. For security, use a different secret than the primary server.
VLAN ID for successfully authenticated devices. This is the main production VLAN for authorized users.
VLAN ID for voice devices (IP phones). This allows separate handling of voice traffic for QoS purposes.
VLAN ID for guest devices or failed authentications. Typically provides limited network access.
VLAN ID when authentication servers are unreachable. Used to maintain basic network access during outages.
The network interface(s) where 802.1X will be enabled. You can specify a single interface, multiple interfaces separated by commas, or a range using a hyphen.

Advanced Options

Allow MAC Authentication Bypass if 802.1X fails. Useful for devices that don't support 802.1X.
Change of Authorization (CoA) allows the RADIUS server to dynamically change authorization parameters (like VLAN) during a session.
Monitor mode allows the switch to perform 802.1X authentication but not enforce it. Useful for testing or gradual deployment.
RadSec (RADIUS over TLS) encrypts RADIUS traffic over TLS, providing stronger security than the standard RADIUS shared secret.
How long (in seconds) to wait before restarting authentication after a failure.
How often (in seconds) to reauthenticate already authenticated clients. Recommended values: 3600-86400 (1-24 hours).
