Cisco IOS 802.1X with MAB

Category: cisco | Created: 2023-06-15

This template configures 802.1X authentication with MAC Authentication Bypass (MAB) fallback on Cisco IOS devices. It includes support for voice VLANs, guest VLANs, and critical VLANs for RADIUS server failure handling.

! Cisco IOS 802.1X with MAB Template
! Created by Dot1Xer Supreme

aaa new-model
aaa authentication dot1x default group radius
aaa authorization network default group radius
aaa accounting dot1x default start-stop group radius

radius server PRIMARY
 address ipv4 10.1.1.100 auth-port 1812 acct-port 1813
 key SecureRadiusKey123
 timeout 5
 retransmit 3

radius server BACKUP
 address ipv4 10.1.1.101 auth-port 1812 acct-port 1813
 key SecureRadiusKey456
 timeout 5
 retransmit 3

aaa group server radius RADIUS-SERVERS
 server name PRIMARY
 server name BACKUP
 deadtime 15

dot1x system-auth-control

vlan 10
 name DATA_VLAN
vlan 20
 name VOICE_VLAN
vlan 30
 name GUEST_VLAN
vlan 999
 name CRITICAL_VLAN

interface GigabitEthernet1/0/1
 switchport mode access
 switchport access vlan 10
 switchport voice vlan 20
 authentication port-control auto
 authentication order dot1x mab
 authentication priority dot1x mab
 authentication event fail action authorize vlan 30
 authentication event server dead action authorize vlan 999
 authentication event server alive action reinitialize
 dot1x pae authenticator
 dot1x timeout tx-period 10
 dot1x max-reauth-req 3
 spanning-tree portfast
 mab

Phased Deployment

Overview

Phased deployment is the recommended approach for implementing 802.1X across most networks. This strategy minimizes disruption by gradually enabling 802.1X in manageable segments.

Recommended Phases

  1. Planning and Preparation (1-2 weeks)
    • Inventory network devices
    • Identify device 802.1X capabilities
    • Configure RADIUS infrastructure
    • Define authentication policies
  2. Phase 1: Monitor Mode (2-4 weeks)
    • Deploy 802.1X in monitor mode across the network
    • Log authentication attempts without enforcement
    • Identify problematic devices and areas
    • Adjust policies based on findings
  3. Phase 2: Low-Impact Areas (2-3 weeks)
    • Enable enforcement in non-critical network segments
    • Implement MAB for devices without 802.1X support
    • Configure guest VLANs for unauthenticated devices
    • Monitor and resolve issues
  4. Phase 3: Expanded Deployment (3-4 weeks)
    • Gradually expand enforcement to more critical areas
    • Refine and update exception policies
    • Conduct user training as needed
    • Establish help desk procedures
  5. Phase 4: Full Enforcement (1-2 weeks)
    • Enable enforcement across all network segments
    • Implement strict authentication policies
    • Fine-tune RADIUS server performance
    • Document final configuration

Benefits

  • Reduces risk by allowing issues to be identified early
  • Provides time to address device compatibility problems
  • Allows IT staff to gain experience gradually
  • Minimizes disruption to business operations
  • Enables policy refinement based on real-world results

Timeline

A typical phased deployment takes 8-12 weeks from planning to full enforcement, depending on network size and complexity.
